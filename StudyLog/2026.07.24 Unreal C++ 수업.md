```cpp file:참조와_BP_노드의_핀과의_상관관계
void Main() {
 int MyNumber, MyNumber2;
```

CPP에서 참조를 매개변수로 넘기는 행위는 주로 여러개의 반환(출력)이 필요할 때 사용한다. 위의 예시를 보면 2개의 변수를 1로 세팅한다. 여기서 유심히 살펴볼 것은 SetTwoNumbersTo1은 입력으로 들어온 참조변수의 값이 중요하지 않다. 어차피 1로 세팅될 것이기 때문에 그 전의 상태는 필요가 없다. 블루 프린트도 이런 관점에서 참조 변수는 입력핀 대신 출력핀만 있는 것이다. 내부에서 값을 세팅 해 주는것을 기본으로 보면 `어떤 참조를 입력으로 넘겨 준 뒤 함수 내부에서 값을 세팅한 후 원래 참조를 사용`하는 방식인데 조금 다르게 생각해보자. 아까도 말했듯 내부에서 값을 세팅하는 함수라고 생각하면 입력의 상태는 중요하지 않다. 그 어떤 값이 들어왔어도 세팅된 값으로 변할 것이다. 그렇다면 왜 BP에서는 입력에 있는 값을 출력으로 보냈을까? 방금 말한 맥락과 비슷하다. 입력의 상태가 중요하지 않으니 아예 입력을 없애버리고 리턴으로 세팅될 값을 반환한다. 이를 코드로 보면 조금 더 명쾌하다.

```cpp file:상황1
void Main() {
 int MyNumber
 SetNumbersTo1(MyNumber, );
}

void SetTwoNumbersTo1(int& FirstOut) {
 FirstOut = 1;
}
```

```cpp file:상황2
void Main() {
 int MyNumber = SetNumbersTo1();
}

void SetTwoNumbersTo1() {
 return 1;
}
```

위 2개 함수는 참조를 넘기고 값을 세팅한다는 관점에서 보면 완벽히 동일하다. 위의 경우는 변수를 선언하고 그 변수의 값을 세팅한 후 사용하는 형태다. 두 번째 경우는 시작부터 세팅된 값을 사용하는 것이다. 이러한 논리로 BP에서 참조 변수의 입력핀은 출력으로 넘어가게 되었다.
하지만 입력된 참조의 상태가 중요할 수 있다. 그런 상황에는 UPARAM() 매크로를 통해서 입력핀을 명시해 줄 수 있다.

```cpp Title:참조를\_입력핀으로\_사용하기
UFUNCTION(BlueprintCallable)
void DoubleTwoNumbers(UPARAM(ref) int32& FirstNumber, UPARAM(ref) int32& SecondNumber) 
{
	FirstNumber = FirstNumber * 2;
	SecondNumber = SecondNumber * 2;
}
```
