# 오늘 배운 내용

## 1. 코딩 규약

### 네이밍

- ==PascalCase==
- ==\_(Underbar)== 사용 지양
- ==변수의 이름==은 ==최대한 자세하게== 적을 것!
- ==정해진 Prefix== 사용

| Prefix | 대상 | 예시 |
|:------:|------|------|
| `A` | Actor | `ACharacter` |
| `U` | UObject / Component | `UCameraComponent` |
| `F` | Struct / 일반 Class | `FVector` |
| `T` | Template | `TArray` |
| `I` | Interface | `IMyInterface` |
| `E` | Enum | `EGameState` |
| `b` | bool | `bIsDead` |

* Coding Standard 읽어보기

## 2. BP와 C++의 관계
- BP는 고수준 
- C++은 저수준
- 상속 관계에서 BP에서 만든 내용은 C++에서 볼 수 없다.

## 4. 자주 사용되는 매크로
- [[UCLASS]]
	- BP에서 클래스 사용하려면 붙이기
	- cpp class를 UObject로 등록해주는 매크로, 다중 상속을 막아줌
- [[USTRUCT]]
	- BP에서 구조체 사용하려면 붙이가
- [[UENUM]]
	- BP에서 열거형 사용 하려면 붙이기
- [[UFUNCTION]]
	- BP에서 보이는 것 뿐 아니라, 구현(+오버라이드)를 BP에서 할 수 있게
- [[UPROPERTY]]
	- BP에서 보이는 것 뿐 아니라, 리플렉션 시스템에 등록할지 (GC, 등)


## 5. 스마트 포인터
- UObject용 스마트 포인터
	- [[TObjectPtr]] : 강한 참조의 스마트 포인터 (hard reference, 바로 로딩됨), 정확히 말하면 GC용 포인터임
	- [[TSoftObjectPtr]] : 에셋 경로를 들고 있어서 바로 로딩되지 않는다. (비동기 로딩 가능)
	- [[TWeakObjectPtr]] : 순환 참조 해결을 위한 포인터 (GC가 수집 시 nullptr로 처리할 수 있다)
- Native
	- [[shared_ptr]]
	- [[unique_ptr]]
	- [[weak_ptr]]


## 6. [[.generated.h]]
- 모든 헤더중에서 항상 마지막에 include할 것
- [[UHT(Unreal Header Tool)]]가 생성한 [[Reflection]] 및 [[UObject]] 관련 코드가 포함된 헤더이다. 
- UCLASS, USTRUCT, UENUM 등에 필요한 메타데이터와 자동 생성 코드가 들어있다.
- 내부에 [[GENENATED_BODY]] 매크로가 사용할 매크로를 작성
	- 정확히 말하면 GENENATED_BODY() -> FID_UnreaCpp_Source_ATestActor_h_15_GENERATED_BODY 이런 식으로 변경되는데, generated.h 내부에 이 변경된 매크로가 사용할 매크로를 적용


## 7. [[UPROPERTY]]
* Keyword
	* VisibleAnywhere: 모든 속성창에 표시
	* VisibleDefaultOnly: 클래스 디폴드에서만 표시
	* VisibleInstanceOnly: 맵에 배치된 액터의 디테일 창에서만
	* EditAnywhere: 어디서나 수정
	* EditDefaultsOnly: 클래스 디폴트에서만 수정
	* EditInstanceOnly: 배치된 액터의 디테일에서만 수정
	* BlueprintReadOnly: BP에서 읽기만
	* BlueprintReadWrite: BP에서 읽기 쓰기 다 가능
키워드들은 중복으로 사용 가능
카테고리 추가하고 싶으면 | 넣고 추가 하면 됨
`UPROPERTY(BlueprintReadWrite, EditAnywhere, Category= "TestActor 변수|Visible")`

## 8. [[UFUNCTION]]
안 붙이면 여러모로 불편해짐
* Keyword
	* BlueprintCallable: cpp에서 작성한 코드를 BP에서 호출 가능
	* BlueprintImplementableEvent: cpp에서 선언만, BP에서 구현
		* 커스텀 이벤트로 만들면 안되고 반드시 오버라이드에서 뽑아 써야함
	* BlueprintNativeEvent: cpp에서 선언 및 구현 가능, BP에서 재정의 가능
		* cpp에서 함수 이름 뒤에 \_Implementation을 붙여서 구현 해야함.
		* 커스텀 이벤트로 만들면 안되고 반드시 오버라이드에서 뽑아 써야함
	* BlueprintPure: 실행핀 없이 호출되는 함수(순서에 상관 없이)(BP에서 녹색으로 표시되는 노드들)
